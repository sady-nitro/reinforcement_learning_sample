# Q-Learning による迷路探索 — 学習手続きの解説

## 1. 前提: 迷路とエージェント

20×20 のグリッド迷路の中を、エージェントがスタート `(1,1)` からゴール `(18,18)` まで移動する問題を扱う。

- **状態 (state)**: エージェントの現在座標 `(row, col)`
- **行動 (action)**: 上・下・左・右の 4 方向
- **制約**: 壁のセルには進入できない。壁に向かって移動すると位置はそのまま

## 2. Q テーブル — 「状態×行動」ごとの価値を記録する表

Q-Learning の核心は **Q テーブル** と呼ばれる表である。

```
q_table = np.zeros((20, 20, 4))
```

- 形状は `(行数 × 列数 × 行動数)` = `(20 × 20 × 4)`
- `q_table[r][c][a]` は「座標 `(r, c)` にいるとき、行動 `a` をとることの"良さ"」を数値で表す
- 初期値はすべて **0**。つまり学習前はどの行動も等しく無価値

このテーブルの値を繰り返し更新していくことで、各マスでどの方向に進むのが最善かをエージェントが「覚えていく」。

## 3. 報酬の設計 — 何が良くて何が悪いか

エージェントが 1 ステップ行動するたびに、環境から **報酬** が返される。

| 状況 | 報酬値 | 意図 |
|------|--------|------|
| ゴールに到達 | **+100.0** | ゴールに辿り着くことを強く動機づける |
| 壁に衝突 (移動失敗) | **-1.0** | 壁にぶつかる行動を避けさせる |
| 通常の 1 歩 | **-0.1** | 無駄に歩き回るほど損になる → 最短経路を促す |

この報酬設計が「ゴールに最短で辿り着くのが最も得」という構造を作っている。

## 4. 1 エピソードの流れ

1 エピソード = 「スタートからゴール到達 (または上限ステップ到達) まで」の一連の試行。
これを 5000 回繰り返す。

```
エピソード開始: エージェントをスタート位置 (1,1) に配置
      │
      ▼
┌───────────────────────────────────┐
│  (A) 行動を選ぶ (ε-greedy)        │
│  (B) 行動を実行し、報酬を受け取る   │
│  (C) Q値を更新する                 │
│  (D) 次の状態に遷移する            │
└──────────┬────────────────────────┘
           │  ゴール到達 or 1000ステップ?
           │    No → ループ先頭 (A) へ
           │    Yes ↓
      エピソード終了
           │
      ε を減衰させる
           │
      次のエピソードへ
```

以下、各ステップを詳しく見る。

### (A) 行動を選ぶ — ε-greedy 方策

```python
if random.random() < epsilon:
    action = random.randrange(NUM_ACTIONS)    # ランダム (探索)
else:
    action = int(np.argmax(q_table[r, c]))    # Q値が最大の行動 (活用)
```

- 確率 ε で **ランダムな行動** を選ぶ (探索: まだ試していない道を開拓)
- 確率 1−ε で **Q値が最大の行動** を選ぶ (活用: これまでの経験で最善と分かっている行動)

**ε の減衰**:

```
初期 ε = 1.0  →  毎エピソード × 0.999  →  下限 0.01
```

| エピソード | ε (概算) | 振る舞い |
|-----------|---------|---------|
| 1 | 1.00 | 100% ランダム。まだ何も知らないので手当たり次第に試す |
| 500 | 0.61 | 約 6 割ランダム。まだ積極的に未知の道を探る |
| 1000 | 0.37 | 約 4 割ランダム。学習した知識を活かしつつ探索も続ける |
| 3000 | 0.05 | ほぼ学習済みの最善手を選ぶが、たまに寄り道してみる |
| 5000 | 0.01 | 99% 最善手。ほぼ最適行動だけをとる |

この「最初は広く試し、徐々に最善手に絞る」仕組みが、未知の良い経路を見逃さずに学習を収束させる鍵となる。

### (B) 行動を実行し、報酬を受け取る

```python
next_state, reward, done = get_reward_and_next(state, action)
```

選んだ方向に 1 マス移動を試みる。環境は「移動後の座標」「報酬」「終了フラグ」を返す。

### (C) Q値を更新する — 学習の核心

ここが Q-Learning のアルゴリズムの根幹部分。

```python
best_next = np.max(q_table[nr, nc])           # 次の状態での最大Q値
q_table[r, c, action] += alpha * (
    reward + gamma * best_next - q_table[r, c, action]
)
```

数式で書くと:

```
Q(s, a) ← Q(s, a) + α × [ r + γ × max Q(s', a') − Q(s, a) ]
                           ~~~~~~~~~~~~~~~~~~~~~~~~~~~
                                   TD誤差
```

| 記号 | 対応する変数 | 値 | 意味 |
|------|------------|-----|------|
| Q(s, a) | `q_table[r, c, action]` | — | 現在の状態 s で行動 a をとることの推定価値 |
| α | `alpha` | 0.1 | 学習率。新しい情報をどれだけ反映するか |
| r | `reward` | +100 / -1 / -0.1 | 今回の行動で得た即時報酬 |
| γ | `gamma` | 0.99 | 割引率。将来の報酬をどれだけ重視するか |
| max Q(s', a') | `best_next` | — | 次の状態 s' で最も良い行動をとった場合の推定価値 |

#### この更新式が意味すること

直感的に言えば:

> **「今回の行動の結果 (即時報酬 + 将来の見通し) が、事前の予想より良ければ Q 値を上げ、悪ければ下げる」**

- `reward + gamma * best_next` = **実際に得た報酬 + 次の状態から先の見込み** (新しい情報)
- `q_table[r, c, action]` = **更新前の予想** (古い情報)
- 両者の差 (**TD 誤差**) に学習率 α を掛けて、少しずつ予想を修正する

#### 割引率 γ = 0.99 の効果

`gamma * best_next` は「一歩先の最大Q値を 0.99 倍して足す」ことを意味する。これが連鎖的に働くため:

- ゴール直前のマス: ゴールで得る +100 の恩恵を直接受ける → Q値が高くなる
- ゴールから 2 歩手前: 1 歩先の高い Q 値が γ 倍されて伝わる → やや高くなる
- ゴールから n 歩手前: 報酬が γ^n 倍に減衰しながら伝わる

つまり **ゴールの「良さ」が波紋のようにゴールから逆方向に伝播していく**。

### (D) 次の状態に遷移する

```python
state = next_state
```

移動先を新たな現在地とし、(A) に戻って次のステップを実行する。

## 5. Q値の伝播 — ゴールから逆算して経路が形成される過程

Q-Learning の学習が進む様子を、具体的な数値例で追ってみる。

### 初期状態 (学習前)

```
すべての Q(s, a) = 0
```

### 最初にゴールに辿り着いたとき

エージェントがランダムに歩き回り、偶然ゴール手前のマス `(17,18)` で「↓ (下)」を選んでゴール `(18,18)` に到達したとする。

```
Q(17,18, ↓) ← 0 + 0.1 × [100 + 0.99 × 0 − 0] = 10.0
```

ゴール手前のマスで「下に行く」の Q 値だけが **10.0** になる。他はすべてまだ 0。

### 次にそのマスを通ったとき

別のエピソードで `(16,18)` から `(17,18)` に移動すると:

```
best_next = max Q(17,18, *) = 10.0  (先ほど更新された値)

Q(16,18, ↓) ← 0 + 0.1 × [-0.1 + 0.99 × 10.0 − 0] = 0.981
```

ゴールから 2 歩手前のマスにも正の Q 値が生まれた。

### これが繰り返されると...

エピソードを重ねるにつれて:

```
ゴール付近:  Q値 = 高い  (報酬源に近い)
     ↑ 伝播
中間地点:    Q値 = 中程度
     ↑ 伝播
スタート付近: Q値 = まだ低い (報酬源から遠い)
```

このように **ゴール側から徐々にスタート方向へ Q 値の「勾配」が形成** されていく。最終的にはスタート地点でも「どの方向に行けばゴールに最短で到達できるか」が Q 値の大小関係として表現される。

### 壁への衝突ペナルティの役割

壁にぶつかると報酬 -1.0 を受け取り、Q 値が下がる。

```
Q(r, c, 壁方向) ← ... + 0.1 × [-1.0 + 0.99 × Q(r,c の最大) − Q(r,c, 壁方向)]
```

何度もぶつかるほどその方向の Q 値は負に沈んでいく。結果、壁に向かう行動は自然に選ばれなくなる。

### ステップペナルティの役割

1 歩ごとに -0.1 の報酬が入るため、遠回りするほど合計報酬が減る。

- 34 歩の最短経路: 報酬 ≈ 100 + 33×(-0.1) = **96.7**
- 100 歩の遠回り: 報酬 ≈ 100 + 99×(-0.1) = **90.1**

Q 値の伝播にもこの差が反映されるため、同じゴールに到達する複数経路のうち **最短のものが自然に最も高い Q 値を持つ** ようになる。

## 6. 学習結果: Q テーブルから最適経路を取り出す

学習が十分に進んだ後、各マスで Q 値が最大の行動を貪欲 (greedy) に選び続けるだけで、最適な経路が得られる。

```python
def extract_path(q_table):
    state = START
    path = [state]
    while state != GOAL:
        action = argmax(q_table[state])    # Q値が最大の方向へ
        state = 移動先
        path.append(state)
    return path
```

これは Q テーブルに刻まれた「Q 値の勾配」を、ゴール方向に下っていく操作に他ならない。

## 7. 学習パラメータのまとめ

| パラメータ | 値 | 役割 |
|-----------|-----|------|
| エピソード数 | 5000 | 試行回数。多いほど Q 値が正確に収束する |
| α (学習率) | 0.1 | 1 回の更新で Q 値をどれだけ変化させるか。大きすぎると不安定、小さすぎると学習が遅い |
| γ (割引率) | 0.99 | 将来の報酬の重み。1 に近いほど遠い将来の報酬も重視する |
| ε (探索率) | 1.0 → 0.01 | ランダム行動の割合。高い間は探索重視、低くなると活用重視 |
| ε 減衰率 | 0.999 | ε を毎エピソード 0.999 倍。約 4600 エピソードで下限 0.01 に到達 |
| 最大ステップ | 1000 | 1 エピソードの上限。ゴールに辿り着けなくても打ち切り |

## 8. 全体像: なぜこれで迷路が解けるのか

```
  ┌──────────────────────────────────────────────────┐
  │  1. ランダムに動き回る (ε が高い)                   │
  │     → たまたまゴールに到達                          │
  │     → ゴール付近の Q 値が上がる                     │
  │                                                    │
  │  2. ゴール付近の Q 値が、隣接マスに波及する          │
  │     → 「ゴールに近づく方向」の Q 値が高くなる        │
  │                                                    │
  │  3. ε が下がり、Q 値の高い方向を選ぶようになる       │
  │     → より効率的にゴールに到達                      │
  │     → 経路全体の Q 値がさらに正確になる              │
  │                                                    │
  │  4. 収束                                           │
  │     → 各マスで最善の方向が確定                      │
  │     → Q テーブルから最短経路を読み出せる             │
  └──────────────────────────────────────────────────┘
```

要約すると、Q-Learning は **「報酬を手がかりに、試行錯誤しながら各状態での最善行動を表に書き込んでいく」** アルゴリズムである。ゴールという報酬源から Q 値の勾配が逆方向に伝播することで、スタートからゴールへの最適経路が自然に浮かび上がる。
